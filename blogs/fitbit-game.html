The use of pulse rate as gaming input presents multiple technological challenges, including:
  • Middleware development
  • Internet protocols
  • Sample rate
Aside of these, it is crucial to develop game mechanics that integrate the technology in a meaningful way.

Two applications were developed to source the data from a Fitbit Versa 2. The first application sits on the watch itself and the second application on the phone that is paired to the watch.

Both applications are programmed in JavaScript and are responsible for collecting data from the watch, beaming the data through Bluetooth to the phone, then from the phone, sending the data to over the Interent through a cloud websocket

The cloud websocket of choice was Pie Socket as they offer a straightforward integration with Unity projects. This is important as the game is developed using the Unity game engine and C#.

Websockets allow for near real-time communication with devices and is often found in machine to machine communication systems. Although this is a game, we still need that machine to machine communication to interact with the sensors in the Fitbit.

Finally, we need to ensure that the communication is coming through at a rate that is fast enough to keep the gaming experience fluid but slow enough that maintains a good level of performance. The sample rate was once every five seconds as received in the game.

With the system development complete, we must focus on how we can turn all of that into a gaming experience that is fun yet challenging. There are two core mechanics in the game:
  • Jog to keep up a good pace against a monster that chases you continously.
  • Sprint over 15 seconds to generate enough boost through your heart rate so you can jump between platforms.
